## Implementation: Retry Loop in HCI_SendDataPkt

### Current Implementation (lines 1323–1354 in hci.c)

The function `HCI_SendDataPkt` in
`$LOCI_TEST_BLE_ROOT/simplelink-lowpower-f3-sdk/source/ti/ble5stack_flash/hci/cc26xx/hci.c`
currently has three branches:

1. If PTM mode is enabled → return `HCI_ERROR_CODE_CONTROLLER_BUSY`
2. If packet length is zero, data pointer is NULL, or the upper byte of
   `connHandle` is non-zero → return `HCI_ERROR_CODE_INVALID_HCI_CMD_PARAMS`
3. Otherwise → call `MAP_LL_TxData(connHandle, pData, pktLen, pbFlag)` and
   return the resulting status

`MAP_LL_TxData` maps to `LL_TxData` via `map_direct.h`. The Link Layer returns
`LL_STATUS_ERROR_UNKNOWN_CONN_HANDLE` (value `0x02`, defined in
`controller/cc26xx/inc/ll.h`) when the connection handle does not correspond to
a known, active connection. This value equals `HCI_ERROR_CODE_UNKNOWN_CONN_ID`
(value `0x02`, defined in `inc/hci.h`).

### Change: Replace the single call with a retry loop

Locate the `else` branch at line 1344 and replace it as follows.

**Before:**

```c
  else // packet is okay
  {
    // attempt to send the packet
    // Note: A return of HCI_SUCCESS from this routine indicates that either
    //       the data was transmitted and freed, or it is still in use
    //       (i.e. queued).
    hciStatus = MAP_LL_TxData( connHandle, pData, pktLen, pbFlag );
  }
```

**After:**

```c
  else // packet is okay
  {
    uint8 retryCount;

    for ( retryCount = 0; retryCount < 3; retryCount++ )
    {
      // attempt to send the packet
      // Note: A return of HCI_SUCCESS from this routine indicates that either
      //       the data was transmitted and freed, or it is still in use
      //       (i.e. queued).
      hciStatus = MAP_LL_TxData( connHandle, pData, pktLen, pbFlag );

      // Retry only when the connection handle was not recognised by the Link
      // Layer.  Any other result (success or a different error) is final.
      if ( hciStatus != HCI_ERROR_CODE_UNKNOWN_CONN_ID )
      {
        break;
      }
    }
    // hciStatus now reflects the outcome of the last (or only) attempt.
  }
```

### Design decisions

- **Retry condition**: Only `HCI_ERROR_CODE_UNKNOWN_CONN_ID` (0x02) triggers a
  retry. This is the same numeric value as `LL_STATUS_ERROR_UNKNOWN_CONN_HANDLE`
  returned by `LL_TxData` when the connection handle is not found in the LL
  connection table. All other status values — including `HCI_SUCCESS` and any
  other error code — break out of the loop immediately.
- **Maximum attempts**: The loop runs at most 3 times (retryCount 0, 1, 2),
  satisfying the requirement of "up to 3 retries before returning an error."
- **Variable type**: `retryCount` is declared as `uint8` to match the type
  conventions used elsewhere in the file for small loop counters.
- **No delay between retries**: No sleep or delay is inserted. The intent is to
  handle a transient race where the LL connection table entry is not yet
  finalized. If a delay were required, it would need an RTOS-aware mechanism
  (e.g., `Task_sleep`) that is outside the scope of this function.
- **No new includes required**: `HCI_ERROR_CODE_UNKNOWN_CONN_ID` is already
  reachable via the existing include chain that provides `hciStatus_t` and all
  other `HCI_ERROR_CODE_*` constants used in this file.
- **All other branches unchanged**: The PTM check, the parameter-validation
  check, the function signature, and the surrounding preprocessor guards
  (`#if defined(CTRL_CONFIG) && (CTRL_CONFIG & (ADV_CONN_CFG | INIT_CFG))`)
  are not modified.
