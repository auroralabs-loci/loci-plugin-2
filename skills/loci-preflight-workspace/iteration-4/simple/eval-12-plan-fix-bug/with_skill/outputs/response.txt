## Plan: Fix unsigned subtraction wraparound in `osal_mem_free`

### Context

The bug is in `osal_mem_free` in `osal_memory.c`. When computing the usable size of a block being freed, the code performs an unsigned subtraction of the header size from the reported block size. If the block header size exceeds the reported size — due to heap metadata corruption or a misaligned pointer — the subtraction wraps to a very large unsigned value. The allocator then treats this garbage region as a huge free block, causing heap corruption.

**Arithmetic:** The defect is an unsigned wraparound on a size subtraction:

```c
/* Existing buggy pattern — pseudocode */
uint16_t freeSize = block->size - OSAL_MEM_HDR_SIZE;
```

- Both `block->size` and `OSAL_MEM_HDR_SIZE` are unsigned (typically `uint16_t` or equivalent).
- If `block->size < OSAL_MEM_HDR_SIZE` (corrupted or misaligned header metadata), the subtraction wraps: e.g., `0 - 4` → `0xFFFC` for a `uint16_t`.
- The wrapped value is used to update the free list, allowing the allocator to claim ownership of memory it does not own — a classic heap-corruption vector.
- Fix: add an explicit underflow guard before the subtraction. If the invariant is violated, treat the block as corrupted and return without modifying the free list.

Arithmetic: ⚠ RISK — unsigned subtraction with no underflow guard; fix must add the guard before the subtraction.

---

**Resources:** `osal_mem_free` inserts the block back into the free list inside a critical section. Two resource concerns arise from the fix:

1. **Critical-section pairing** — the new early-return path introduced by the guard must call `HAL_EXIT_CRITICAL_SECTION` before returning. A missing exit would leave the system in a permanently locked state (deadlock on the next interrupt-masked path).
2. **Block leak on corrupt metadata** — if the guard fires and returns early, the block is not re-inserted into the free list. This is the correct safe behavior when metadata is corrupt (leaking one block is preferable to corrupting the entire heap), but it should be documented at the guard site.

Resources: ⚠ RISK — the new early-return path must call `HAL_EXIT_CRITICAL_SECTION` before returning; omitting it would deadlock the system.

---

**Execution fit: ADJUST PLAN**

→ Add a pre-subtraction underflow guard and ensure `HAL_EXIT_CRITICAL_SECTION` is called on every return path, including the new early-return introduced by the guard.

### Adjusted plan (ordered steps)

1. **Read the file** — open `osal_memory.c` and locate `osal_mem_free`. Identify:
   - The exact unsigned subtraction expression computing free size.
   - Where `HAL_ENTER_CRITICAL_SECTION` is called and what variable name it uses (e.g., `intState`).
   - All existing return points, to confirm the critical-section exit pattern.

2. **Add the underflow guard before the subtraction** — immediately before the line that computes the free size, insert:

   ```c
   if ( block->size < OSAL_MEM_HDR_SIZE )
   {
     /* Corrupted block header: size field is smaller than the header itself.
        Performing the subtraction would wrap to a huge unsigned value and
        corrupt the free list. Exit the critical section and return safely. */
     HAL_EXIT_CRITICAL_SECTION( intState );
     return;
   }
   ```

   The guard must be placed *after* `HAL_ENTER_CRITICAL_SECTION` so that the exit call is correctly paired.

3. **Verify all exit paths** — after adding the guard, trace every `return` in the function and confirm each one is preceded by `HAL_EXIT_CRITICAL_SECTION` (or occurs before the `ENTER` call).

4. **Add a debug assertion** (optional but recommended) — in debug builds, trigger a louder diagnostic before the guard:

   ```c
   OSAL_ASSERT( block->size >= OSAL_MEM_HDR_SIZE );
   ```

   This surfaces corruption in development while the release build fails safely.

5. **Review call sites** — check the callers of `osal_mem_free` to see whether they validate the pointer before calling. The guard prevents heap corruption but does not identify the root cause (the code that produced the bad block header).

6. **Test** — verify with a unit test that passes a block whose `block->size < OSAL_MEM_HDR_SIZE` and confirms the function returns without modifying the free list and without deadlocking (critical section properly exited).
