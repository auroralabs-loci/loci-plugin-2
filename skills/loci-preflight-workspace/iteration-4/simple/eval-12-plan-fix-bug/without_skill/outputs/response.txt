## Plan: Fix unsigned subtraction wrap in osal_mem_free

### Root Cause

In `osal_mem_free` (located at `$LOCI_TEST_BLE_ROOT/simplelink-lowpower-f3-sdk/source/ti/ble5stack_flash/osal/src/common/osal_memory.c`), the free block size is computed with an expression of the form:

```c
uint16_t freeSize = pHdr->size - OSAL_MEM_HDR_SIZE;
```

Both operands are unsigned. If `pHdr->size` is less than `OSAL_MEM_HDR_SIZE` (due to heap corruption or a caller supplying an invalid pointer), the subtraction wraps around to a value near `UINT16_MAX` or `SIZE_MAX`. Any subsequent use of that wrapped value — for memset, coalescing, or free-list insertion — will operate on a huge number and corrupt the heap.

---

### Step 1 — Read the current implementation

Before changing anything, read `osal_mem_free` in `osal_memory.c` to:
- Confirm the exact types of `pHdr->size` and `OSAL_MEM_HDR_SIZE`.
- Identify all unsigned subtractions involving block size and header overhead.
- Determine which error path is appropriate for the target build (assert, early return, or corruption flag).

---

### Step 2 — Add an underflow guard before the subtraction

After reading the code, add an explicit check immediately before the subtraction:

```c
if (pHdr->size < OSAL_MEM_HDR_SIZE)
{
    HAL_ASSERT_FORCED();   /* debug trap — heap is corrupt */
    return;                /* release build: skip the free silently */
}
uint16_t freeSize = pHdr->size - OSAL_MEM_HDR_SIZE;
```

The assert traps immediately in debug builds. The `return` after it ensures release builds do not proceed with the wrapped value.

---

### Step 3 — Handle the zero-size boundary case

When `pHdr->size == OSAL_MEM_HDR_SIZE`, `freeSize` is zero. Verify that the free-list insertion and any coalescing code handle a zero-length payload block gracefully (no infinite loops, no zero-size coalescing that corrupts neighbours).

---

### Step 4 — Audit similar patterns in related helpers

Search for other unsigned subtractions involving `OSAL_MEM_HDR_SIZE` in:
- `osal_mem_alloc`
- Block coalescing / merge helpers
- Any other size arithmetic in the same file

Apply the same guard pattern to each site.

---

### Step 5 — Test

Add targeted tests:
- **Corrupt header**: call `osal_mem_free` with a block whose `size` field has been set to a value less than `OSAL_MEM_HDR_SIZE`; confirm the assert fires in debug and no crash occurs in release.
- **Boundary**: `pHdr->size == OSAL_MEM_HDR_SIZE` — confirm zero-length free completes without corruption.
- **Normal free**: standard alloc/free cycle — confirm no regression.

Run under UBSan/ASan if the toolchain supports it to catch any remaining unsigned wrap sites.
