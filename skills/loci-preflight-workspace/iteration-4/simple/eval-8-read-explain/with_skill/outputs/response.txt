GAP_DeviceInit in the TI SimpleLink BLE5 stack (gap.c) initializes the Generic Access Profile layer for the device. The function signature is:

    bStatus_t GAP_DeviceInit(uint8_t profileRole,
                             uint8_t taskID,
                             GAP_Addr_Modes_t addrMode,
                             uint8_t* pRandomAddr);

Parameters:

1. profileRole (uint8_t)
   Bitmask that specifies which BLE roles this device will take on. Common values are:
   - GAP_PROFILE_BROADCASTER  — advertises only, no connections
   - GAP_PROFILE_OBSERVER     — scans only, no connections
   - GAP_PROFILE_PERIPHERAL   — advertises and accepts connections (slave role)
   - GAP_PROFILE_CENTRAL      — scans and initiates connections (master role)
   Multiple roles can be OR'd together if the device needs to support more than one
   simultaneously (subject to hardware and stack limitations).

2. taskID (uint8_t)
   The OSAL (Operating System Abstraction Layer) task ID of the application task that
   will own GAP events. When the GAP layer needs to deliver an event to the application
   (for example, a GAP_DEVICE_INIT_DONE_EVENT after initialization completes, or a
   connection event), it sends the event message to this task. You get the task ID by
   calling OSAL_RegisterTask or by receiving it in your application task's init function.

3. addrMode (GAP_Addr_Modes_t)
   Specifies what kind of Bluetooth device address the local device will use during
   advertising and connection setup. The enumerated values include:
   - ADDRMODE_PUBLIC             — use the IEEE-assigned 48-bit public address burned
                                   into the device
   - ADDRMODE_RANDOM             — use a static random address (set once at startup,
                                   stays the same until next power cycle)
   - ADDRMODE_RP_WITH_PUBLIC_ID  — resolvable private address (RPA) with a public
                                   identity address; the RPA rotates periodically so
                                   passive observers cannot track the device, but bonded
                                   peers with the IRK can resolve it back to the public
                                   identity
   - ADDRMODE_RP_WITH_RANDOM_ID  — resolvable private address with a random static
                                   identity address; same rotation behavior as above but
                                   the identity address is itself random
   - ADDRMODE_NOTAVAIL           — address not yet configured (rarely used at init time)
   The address mode directly affects privacy behavior and which identity address is
   advertised in the AD structures or used during pairing.

4. pRandomAddr (uint8_t*)
   Pointer to a 6-byte buffer holding the random static address to load into the
   controller. This parameter is only meaningful when addrMode is ADDRMODE_RANDOM or
   one of the ADDRMODE_RP_WITH_RANDOM_ID variants. If the address mode does not require
   a random address (e.g., ADDRMODE_PUBLIC), pass NULL. If you pass a non-NULL pointer
   with a public address mode, the stack ignores the value. The buffer must remain valid
   through the initialization call; the stack copies the bytes into the controller, so
   the caller does not need to keep the buffer alive afterward.

Return value:
   SUCCESS (0) if the request was accepted and queued; the actual completion is
   signaled asynchronously via a GAP_DEVICE_INIT_DONE_EVENT sent to the taskID task.
   A non-zero status (e.g., INVALIDPARAMETER, bleAlreadyInRequestedMode) means the
   call was rejected synchronously before any controller interaction.

Initialization flow summary:
   GAP_DeviceInit is the first GAP call the application must make. It configures the
   role, registers the callback task, and programs the controller with the chosen
   address mode and optional random address. Until the GAP_DEVICE_INIT_DONE_EVENT
   arrives, no other GAP or HCI advertising/scanning/connection calls should be issued.
   After the event arrives, the stack is ready and the application can proceed with
   GAP_SetParamValue calls, GAPRole_StartDevice, or equivalent role-specific setup.
