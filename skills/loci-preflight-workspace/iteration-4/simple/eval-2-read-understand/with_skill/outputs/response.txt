`GAPBondMgr_FindAddr` is a function in the TI BLE5 stack's GAP Bond Manager (`gapbondmgr.c`) that searches the bonding database for a stored bond record matching a given Bluetooth device address. Its purpose is to determine whether a particular peer device has a pre-existing bond with the local device — meaning they have previously completed pairing and exchanged long-term keys.

The function iterates over the bond table entries held in non-volatile (NV) storage and compares each stored address against the provided address and address type. If a match is found, the function returns success and populates an output parameter with the index of the matching bond record. If no match is found, it returns a failure status and the output index is left unset or set to an invalid value.

Parameters:

1. `pDevAddr` — A pointer to a 6-byte Bluetooth device address (BD_ADDR type) identifying the peer device to look up. This is the address that will be compared against the addresses stored in each bond table slot.

2. `addrType` — The address type of the peer device (for example, `ADDRTYPE_PUBLIC`, `ADDRTYPE_RANDOM`, or a resolvable/non-resolvable private address type). Because the same 6-byte address value can refer to different devices depending on address type, this parameter is required alongside `pDevAddr` to form a unique peer identity.

3. `pIdx` — An output parameter: a pointer to a `uint8_t` (or similar small integer type) that is written with the bond table index of the matching record when the function returns success. The caller uses this index to subsequently read or update the full bond record from NV storage (for example, to retrieve the LTK, IRK, or CCCD configuration for that peer).
