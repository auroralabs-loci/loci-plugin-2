## Adding GAPBondMgr_FindAddr as the first check in OAD_open

### Overview

The goal is to add `GAPBondMgr_FindAddr` as the very first check inside `OAD_open` in
`$LOCI_TEST_BLE_ROOT/simplelink-lowpower-f3-sdk/source/ti/ble5stack_flash/profiles/oad/cc26xx/oad.c`
so that OAD rejects image transfers from unbonded devices before any setup is performed.

### Current OAD_open signature and body (lines 424–512)

```c
uint8_t OAD_open(uint32_t oadTimeout)
{
    uint8_t status = OAD_SUCCESS;
    uint8_t oadCCCDTableSize = (sizeof(gattCharCfg_t) * linkDBNumConns)
                                 * OAD_CCCD_CNT;

    oadCCCDTable = (gattCharCfg_t *)ICall_malloc(oadCCCDTableSize);
    if (oadCCCDTable == NULL)
    {
        return (OAD_NO_RESOURCES);
    }
    // ... rest of setup ...
}
```

### Required change

**Step 1 — Ensure `gapbondmgr.h` is included**

`oad.c` already includes `icall_ble_api.h`, which provides the `GAPBondMgr_FindAddr` macro via
`icall_ble_api.h` (line 57). No additional include is needed.

**Step 2 — Obtain the peer address**

`OAD_open` does not currently receive a connection handle or peer address as a parameter.
The peer address and address type must be retrieved from the link database for the active
connection using `linkDB_GetInfo`. The active connection handle needs to be passed in or
accessed via a module-level variable. For this change, assume the OAD module stores the
current connection handle (e.g., `oadConnHandle`) set when the OAD write characteristic
callback fires.

**Step 3 — Insert the bonding check as the first statement**

Replace the opening of `OAD_open` as follows:

**Before:**

```c
uint8_t OAD_open(uint32_t oadTimeout)
{
    uint8_t status = OAD_SUCCESS;
    uint8_t oadCCCDTableSize = (sizeof(gattCharCfg_t) * linkDBNumConns)
                                 * OAD_CCCD_CNT;
```

**After:**

```c
uint8_t OAD_open(uint32_t oadTimeout)
{
    /* Reject unbonded devices before any OAD setup. */
    linkDBInfo_t linkInfo;
    if (linkDB_GetInfo(oadConnHandle, &linkInfo) != SUCCESS)
    {
        return OAD_NO_RESOURCES;
    }

    uint8_t bondIdx;
    if (GAPBondMgr_FindAddr(linkInfo.addr,
                             (GAP_Peer_Addr_Types_t)(linkInfo.addrType & MASK_ADDRTYPE_ID),
                             &bondIdx,
                             NULL,
                             NULL) != SUCCESS)
    {
        return OAD_NO_RESOURCES;
    }

    uint8_t status = OAD_SUCCESS;
    uint8_t oadCCCDTableSize = (sizeof(gattCharCfg_t) * linkDBNumConns)
                                 * OAD_CCCD_CNT;
```

### Notes

- `MASK_ADDRTYPE_ID` masks off the privacy bit to get the identity address type, consistent
  with how other callers of `GAPBondMgr_FindAddr` in the SDK use it (e.g., in `gapbondmgr.c`).
- `oadConnHandle` should be set to the current connection handle in the OAD write callback
  before `OAD_open` is called. If no such module-level variable exists yet, it should be
  added to the OAD module's static state.
- By placing this check before the `ICall_malloc` call for `oadCCCDTable`, no heap memory is
  allocated for connections that will be rejected. The early-return path is clean with no
  resources to free.
- If a more descriptive return code is desired for the caller to distinguish "not bonded"
  from "out of resources", define a new constant (e.g., `OAD_BONDING_REQUIRED`) in `oad.h`
  and return that instead of `OAD_NO_RESOURCES`.
