### Existing function signature

File: `/home/melisa/BLE/simplelink-lowpower-f3-sdk/source/ti/ble5stack_flash/host/gapbondmgr.c`, line 992:

```c
bStatus_t GAPBondMgr_FindAddr(uint8_t *pDevAddr,
                              GAP_Peer_Addr_Types_t addrType,
                              uint8_t *pIdx,
                              GAP_Peer_Addr_Types_t *pIdentityAddrType,
                              uint8_t *pIdentityAddr)
```

`GAPBondMgr_FindAddr` already accepts `pIdx` as an optional output parameter — when non-NULL, it is set to the bond index of the matched entry. Many internal callers already use this: e.g., lines 1852, 2690, 4342, 4854 pass `&idx` or `&bondIdx` as `pIdx`. Callers that don't need the index pass NULL (e.g., line 4303).

The function returns `bStatus_t` (SUCCESS, INVALIDPARAMETER, or bleGAPNotFound). There is no separate "bond index" in the return value; it is already the `pIdx` output parameter.

### Interpretation

The task asks to "add a method" that exposes the bond index as an output parameter. Since the existing function already provides this, the most sensible interpretation is to add a new companion function — a simpler wrapper — that makes the bond-index output mandatory (non-optional) and presents a cleaner interface for callers that only care about the bond index and not the full identity address resolution:

New function: `GAPBondMgr_FindAddrGetIdx`

/**
 * @brief  Search for an address in the bonding table and return the bond index.
 *
 * Convenience wrapper around GAPBondMgr_FindAddr that requires the bond index
 * output parameter to be non-NULL. Use this function when the bond index is
 * always needed and the identity address details are not required.
 *
 * @param pDevAddr  peer's address (must not be NULL)
 * @param addrType  peer's address type
 * @param pBondIdx  pointer to byte to receive the bond index (must not be NULL)
 *
 * @return SUCCESS if the address was found or resolved to an address in the
 *         bonding table
 * @return INVALIDPARAMETER if pDevAddr or pBondIdx is NULL
 * @return bleGAPNotFound if the address was not found
 */
extern bStatus_t GAPBondMgr_FindAddrGetIdx(uint8_t *pDevAddr,
                                           GAP_Peer_Addr_Types_t addrType,
                                           uint8_t *pBondIdx);
```

### Step 2: Add the implementation to the .c file

File: `/home/melisa/BLE/simplelink-lowpower-f3-sdk/source/ti/ble5stack_flash/host/gapbondmgr.c`

/*********************************************************************
 * @brief   Resolve an address from bonding information and return the
 *          bond index as a required output parameter.
 *
 * Public function defined in gapbondmgr.h.
 */
bStatus_t GAPBondMgr_FindAddrGetIdx(uint8_t *pDevAddr,
                                    GAP_Peer_Addr_Types_t addrType,
                                    uint8_t *pBondIdx)
{
  if (pBondIdx == NULL)
  {
    return (INVALIDPARAMETER);
  }

  return GAPBondMgr_FindAddr(pDevAddr, addrType, pBondIdx, NULL, NULL);
}
```
