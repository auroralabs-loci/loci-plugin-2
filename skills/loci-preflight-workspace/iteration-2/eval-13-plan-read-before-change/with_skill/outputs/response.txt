**How GAP_DeviceInit validates its addrMode parameter:**

All addrMode validation runs inside `GAP_DeviceInit_validate_params()` (gap.c, ~lines 107–221), called at the top of `GAP_DeviceInit()`.

**Step 1 — Already-initialized guard (runs before addrMode is checked)**

If `MAP_gapGetState() == GAP_INITSTATE_READY`, the function returns `bleIncorrectMode` immediately — addrMode is irrelevant.

**Step 2 — Range check**

The `GAP_Addr_Modes_t` enum has four legal values (0x00–0x03). Any value `> ADDRMODE_RP_WITH_RANDOM_ID` (0x03) returns `INVALIDPARAMETER`. Also: if addrMode is `ADDRMODE_RANDOM` (0x01) or `ADDRMODE_RP_WITH_RANDOM_ID` (0x03) and `pRandomAddr == NULL`, that also returns `INVALIDPARAMETER`.

**Step 3 — Random Static Address format check**

Applies only when addrMode is `ADDRMODE_RANDOM` or `ADDRMODE_RP_WITH_RANDOM_ID`. The supplied address is rejected (`INVALIDPARAMETER`) if:
- Lower 46 bits are all zero
- All 48 bits are 1 (0xFF:FF:FF:FF:FF:FF)
- Top 2 bits of byte[5] are not `0b11` — checked via `GAP_IS_ADDR_RS()` using `RANDOM_ADDR_HDR_MASK = 0xC0` and `STATIC_ADDR_HDR = 0xC0`

`ADDRMODE_PUBLIC` and `ADDRMODE_RP_WITH_PUBLIC_ID` skip this check entirely.
