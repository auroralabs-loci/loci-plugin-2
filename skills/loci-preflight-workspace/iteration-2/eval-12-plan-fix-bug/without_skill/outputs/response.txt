## Plan: Fix unsigned subtraction wrap in osal_mem_free

### Root Cause

`osal_mem_free` computes the usable size of a freed block with an expression like:

```c
uint16_t freeSize = pHdr->size - OSAL_MEM_HDR_SIZE;
```

Both operands are unsigned. If `pHdr->size` is smaller than `OSAL_MEM_HDR_SIZE` (due to heap corruption or a caller bug), the subtraction wraps to a value near `SIZE_MAX`, corrupting the free-list.

### Step 1 — Locate the affected code

Find `osal_mem_free` in `source/ti/ble5stack/osal/src/common/osal_memory.c` or `osal_memory_icall.c`. Identify the exact type of the size field and header-overhead constant.

### Step 2 — Add a guard before the subtraction

**Preferred (release-safe):**
```c
if (pHdr->size < OSAL_MEM_HDR_SIZE)
{
    HAL_ASSERT_FORCED();   /* trap in debug */
    return;
}
uint16_t freeSize = pHdr->size - OSAL_MEM_HDR_SIZE;
```

### Step 3 — Audit similar patterns

Search for other unsigned subtractions involving `OSAL_MEM_HDR_SIZE` in alloc, merge, and coalesce helpers. Apply the same guard to each site.

### Step 4 — Handle zero-size blocks

After the fix, `freeSize` could be `0` when `pHdr->size == OSAL_MEM_HDR_SIZE`. Verify free-list insertion handles this gracefully.

### Step 5 — Test

Add unit tests: corrupt-header (size < HDR_SIZE), normal free, boundary (size == HDR_SIZE). Run under UBSan/ASan if available.
