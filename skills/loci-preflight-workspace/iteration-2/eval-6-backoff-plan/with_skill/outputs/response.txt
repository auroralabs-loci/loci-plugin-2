## Preflight: connection parameter update retry (exponential backoff)

Call graph:  OK — the backoff is self-contained arithmetic inside an existing retry loop; no new function declarations are needed. The existing `os_delay_ms` / timer call is already in scope.

Arithmetic:  ⚠ RISK — `interval *= 2` can overflow before the cap check runs. If `interval` is `uint32_t` and starts at e.g. 5000 ms, after 13 doublings it exceeds `UINT32_MAX`. The cap check `if (interval > 10000) interval = 10000` is never reached because the multiply already wrapped.

Fix: guard the multiply before it happens:
```c
if (interval <= CAP_MS / 2) {
    interval *= 2;
} else {
    interval = CAP_MS;
}
```
Also ensure the type is `uint32_t` (not `int` or `uint16_t`) and the cap constant uses the `U` suffix: `#define CONN_PARAM_RETRY_CAP_MS 10000U`.

Resources:   OK — no new heap allocation, locks, or resource ownership changes.

Execution fit: ADJUST PLAN
→ Use `uint32_t`, apply the overflow-safe pre-multiply cap guard shown above before writing.
