GAP_DeviceInit typically takes the following parameters:

1. taskID (uint8_t) — The OSAL task ID assigned to the GAP layer. This tells the GAP module which task to post events to when asynchronous operations complete. It must match the task registered with OSAL before calling this function.

2. profileRole (uint8_t) — A bitmask indicating the Bluetooth role(s) the device will assume. Common values include:
   - GAP_PROFILE_BROADCASTER — advertise only, no connections
   - GAP_PROFILE_OBSERVER — scan only, no connections
   - GAP_PROFILE_PERIPHERAL — connectable, accepts incoming connections
   - GAP_PROFILE_CENTRAL — initiates connections to peripherals
   Multiple roles can be OR'd together if the stack build supports it.

3. pAppCallbacks (gapBondCBs_t * or similar callback struct pointer) — A pointer to a struct of application-level callback functions that GAP will invoke for security/bonding events. Passing NULL disables those callbacks.

Initialization flow: GAP_DeviceInit is called once during startup after OSAL task init. It registers the task ID and role, configures the controller, and returns a GAP_DeviceInitDone_Event asynchronously. No other GAP calls that depend on an initialized device should be made before that event is received.
