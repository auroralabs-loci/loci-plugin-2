GAP_DeviceInit takes the following parameters:

1. **taskID** (`uint8_t`) — The OSAL task ID for the GAP layer. GAP posts asynchronous events (like GAP_DeviceInitDone) to this task. It must match the ID registered with OSAL before calling this function.

2. **profileRole** (`uint8_t`) — A bitmask of the Bluetooth role(s) the device will assume:
   - `GAP_PROFILE_BROADCASTER` — advertise only
   - `GAP_PROFILE_OBSERVER` — scan only
   - `GAP_PROFILE_PERIPHERAL` — connectable peripheral
   - `GAP_PROFILE_CENTRAL` — central, initiates connections
   Multiple roles can be OR'd together if the stack build supports it.

3. **maxScanResponses** (`uint8_t`) — (Central/Observer only) Maximum number of scan responses stored before GAP_DeviceDiscoveryDoneEvent is fired.

4. **pIRK** (`uint8_t *`) — Pointer to a 16-byte Identity Resolving Key, or NULL to use a default/auto-generated key.

5. **pSRK** (`uint8_t *`) — Pointer to a 16-byte Signature Resolving Key, or NULL.

6. **signCounter** (`uint32_t *`) — Pointer to the sign counter for authenticated data signing, or NULL.

**Initialization flow:** Call once during startup after OSAL task init. GAP registers the task ID, role, and security keys, then configures the BLE controller. It returns a `GAP_DeviceInitDone_Event` asynchronously. Do not make other GAP calls that require an initialized device before receiving that event.
