The function OAD_processQueue is the main event-processing loop for Over-the-Air Download. It dequeues OAD_msg_t messages and dispatches them based on type:

- OAD_WRITE_IDENTIFY_REQ → validates the incoming image header (size, CRC, version)
- OAD_WRITE_BLOCK_REQ → writes the received block to flash and advances the block counter
- OAD_IMAGE_COMPLETE → verifies the final CRC and reboots into the new image
- OAD_TIMEOUT → handles stalled transfers

Each branch reads from the message, performs its work, and frees the buffer. The function returns after processing one message per call; the caller is responsible for re-posting the event if more messages remain in the queue.
