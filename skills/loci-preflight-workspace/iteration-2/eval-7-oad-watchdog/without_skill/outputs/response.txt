## Adding a Watchdog Reset for Stalled OAD Queue in OAD_processQueue

### Overview of the Existing Code

`OAD_processQueue` (oad.c ~line 629) processes messages from a POSIX message queue (`hOadQ`). The existing inactivity mechanism is `oadActivityClk` — a TI RTOS Clock that fires `oadInactivityTimeout` after `oadStateTimeout` ms (default 10,000 ms), triggering `oadResetState()` — an OAD-level session reset only, not a hardware device reset.

### Design: Use TI Drivers Watchdog API

Use `<ti/drivers/Watchdog.h>`:
- `Watchdog_open(index, params)` — arms the hardware watchdog
- `Watchdog_clear(handle)` — resets the countdown ("pet") on each queue iteration
- `Watchdog_setReload(handle, ticks)` — sets 30-second timeout

### Step-by-Step Changes

**1. Add static handle (~line 328):**
```c
static Watchdog_Handle oadWatchdogHandle = NULL;
```

**2. Add `#include <ti/drivers/Watchdog.h>` to includes.**

**3. Open and configure in `OAD_open` after queue creation:**
```c
Watchdog_Params wdParams;
Watchdog_Params_init(&wdParams);
wdParams.resetMode = Watchdog_RESET_ON;
oadWatchdogHandle = Watchdog_open(CONFIG_WATCHDOG0, &wdParams);
if (oadWatchdogHandle != NULL) {
    uint32_t ticks = Watchdog_convertMsToTicks(oadWatchdogHandle, 30000);
    if (ticks != 0) {
        Watchdog_setReload(oadWatchdogHandle, ticks);
    }
}
```

**4. Clear watchdog at top of `while (msgSize > 0)` loop in `OAD_processQueue`:**
```c
while (msgSize > 0) {
    if (oadWatchdogHandle != NULL) {
        Watchdog_clear(oadWatchdogHandle);
    }
    // existing dispatch...
```

### Critical Caveats

1. **One watchdog per device:** Audit all `Watchdog_open` calls; a second open returns NULL.
2. **Cannot be disabled:** On CC23xx, once started, use `Watchdog_setReload(handle, 0xFFFFFFFF)` to disarm when OAD is idle.
3. **30-second period is within hardware limits:** CC23xx watchdog is 32-bit at 32 kHz — `convertMsToTicks(handle, 30000)` will not overflow.
4. **Mid-download reset safety:** Verify `oadCheckDL` and `oadValidateCandidateHdr` properly reject truncated images on next boot.
5. **Keep `debugStallMode = Watchdog_DEBUG_STALL_ON`** so breakpoints don't trigger the watchdog.
