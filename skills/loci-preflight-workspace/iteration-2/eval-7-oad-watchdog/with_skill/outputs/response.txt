## Preflight: OAD_processQueue (watchdog addition)

Call graph:  ⚠ RISK — Adding `Watchdog_open` inside `OAD_open` introduces a new dependency on `CONFIG_WATCHDOG0`. If another module has already called `Watchdog_open` on the same index, the call returns NULL and stall detection silently fails with no error. Also, `Watchdog_clear` will be called from within the task context of whoever calls `OAD_processQueue` — confirm this is a task context and not an ISR/SWI (the API is not reentrant from interrupt context per TI docs).

Arithmetic:  OK — `Watchdog_convertMsToTicks(handle, 30000)` returns a `uint32_t`. The CC23xx watchdog is 32-bit at 32 kHz; maximum period is ~131,000 seconds, so 30,000 ms (30 s) produces ~983,040 ticks — well within range, no overflow.

Resources:   ⚠ RISK — The TI CC23xx watchdog cannot be stopped once started (`Watchdog_RESET_ON` mode). `OAD_close` must call `Watchdog_setReload(handle, 0xFFFFFFFF)` to effectively disarm it, not `Watchdog_close`. If `OAD_open` fails after the watchdog is opened (e.g., GATT registration fails), the watchdog must still be disarmed in the error path — currently OAD_open has no cleanup on failure (a separate issue noted in the code review). Every code path out of `OAD_open` after watchdog init must either clear-on-schedule or reload to max.

Execution fit: ADJUST PLAN
→ Add a watchdog-open audit (assert no other opener), add disarm-on-close in `OAD_close`, and add disarm in `OAD_open`'s error paths.

## Adjusted Plan

### Step 1 — Audit watchdog ownership
Search for all `Watchdog_open` calls in the project. If another module uses `CONFIG_WATCHDOG0`, either use a different index or add a shared ownership protocol.

### Step 2 — Add static handle and include
```c
#include <ti/drivers/Watchdog.h>
static Watchdog_Handle oadWatchdogHandle = NULL;
```

### Step 3 — Open and configure in `OAD_open` (after queue creation, before returning)
```c
Watchdog_Params wdParams;
Watchdog_Params_init(&wdParams);
wdParams.resetMode = Watchdog_RESET_ON;
oadWatchdogHandle = Watchdog_open(CONFIG_WATCHDOG0, &wdParams);
if (oadWatchdogHandle != NULL) {
    uint32_t ticks = Watchdog_convertMsToTicks(oadWatchdogHandle, 30000);
    if (ticks != 0) {
        Watchdog_setReload(oadWatchdogHandle, ticks);
    }
}
```

Add `Watchdog_setReload(oadWatchdogHandle, 0xFFFFFFFF)` to all early-return error paths in `OAD_open`.

### Step 4 — Clear watchdog in `OAD_processQueue` dispatch loop
```c
while (msgSize > 0) {
    if (oadWatchdogHandle != NULL) {
        Watchdog_clear(oadWatchdogHandle);
    }
    // existing dispatch...
```

### Step 5 — Disarm in `OAD_close`
```c
if (oadWatchdogHandle != NULL) {
    Watchdog_setReload(oadWatchdogHandle, 0xFFFFFFFF);
    oadWatchdogHandle = NULL;
}
```

### Step 6 — Verify mid-download reset safety
Confirm `oadCheckDL` and `oadValidateCandidateHdr` properly reject truncated images on next boot before merging.
