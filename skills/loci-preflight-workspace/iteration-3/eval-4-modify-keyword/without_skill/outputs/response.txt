## Understanding `Connection_getConnIndex`

In the TI BLE5Stack SDK, `Connection_getConnIndex` typically looks like this:

```c
uint8_t Connection_getConnIndex(uint16_t connHandle)
{
    uint8_t i;
    for (i = 0; i < MAX_NUM_BLE_CONNS; i++)
    {
        if (connList[i].connHandle == connHandle)
        {
            return i;
        }
    }
    return 0; // Bug: 0 is also a valid index
}
```

The problem is that returning `0` when no connection is found is ambiguous — index `0` is a legitimate connection slot. Callers cannot distinguish "connection is at index 0" from "connection not found."

## Required Changes

### 1. Change the return type from `uint8_t` to `int8_t`

The function signature must be updated so it can represent `-1`:

**In `app_connection.c`:**
```c
// Before:
uint8_t Connection_getConnIndex(uint16_t connHandle)

// After:
int8_t Connection_getConnIndex(uint16_t connHandle)
```

**In the corresponding header (typically `app_connection.h`):**
```c
// Before:
uint8_t Connection_getConnIndex(uint16_t connHandle);

// After:
int8_t Connection_getConnIndex(uint16_t connHandle);
```

### 2. Change the not-found return value from `0` to `-1`

```c
int8_t Connection_getConnIndex(uint16_t connHandle)
{
    uint8_t i;
    for (i = 0; i < MAX_NUM_BLE_CONNS; i++)
    {
        if (connList[i].connHandle == connHandle)
        {
            return (int8_t)i;
        }
    }
    return -1; // Clearly means "not found"
}
```

### 3. Update all callers

Every call site that currently checks `== 0` or uses the result as an index must be updated. The typical pattern is:

```c
// Before (incorrect check):

if (connIndex == MAX_NUM_BLE_CONNS) { /* not found */ }
// or simply using it without checking

// After (correct check):
int8_t connIndex = Connection_getConnIndex(connHandle);
if (connIndex == -1)
{
    // Handle not-found case
    return;
}
// Now safe to use connIndex as an array index
```

Search for all usages of `Connection_getConnIndex` in the project (especially in `app_connection.c` itself and any files that include `app_connection.h`) and update each one.

### 4. Update any local variable types that store the return value

Any variable declared as `uint8_t` that receives the return value of this function should be changed to `int8_t`:
