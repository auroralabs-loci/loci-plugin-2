## Preflight Report

**Target file:** `$LOCI_TEST_BLE_ROOT/simplelink-lowpower-f3-sdk/source/ti/ble5stack_flash/osal/src/common/osal_memory.c`
**Function:** `osal_mem_free`
**Bug class:** Unsigned integer underflow / wraparound

### Arithmetic Check — Unsigned Wraparound Risk

In `osal_mem_free`, the amount of memory being returned to the free pool is computed roughly as:

```c
osalMemHdr_t *hdr = (osalMemHdr_t *)((uint8_t *)ptr - OSAL_MEM_HDRSZ);
uint16_t freeSize = hdr->size - OSAL_MEM_HDRSZ;
```

Both `hdr->size` and `OSAL_MEM_HDRSZ` are unsigned types (typically `uint16_t` and a `uint8_t`-sized constant). If the stored `hdr->size` is less than `OSAL_MEM_HDRSZ` — which can happen due to heap corruption, a double-free, or a misaligned pointer — the subtraction wraps around to a very large value (e.g., `0xFFFE` or similar), causing the allocator to believe it is reclaiming a huge block. This corrupts the free-list and can cause subsequent allocations to overlap live data or crash.

**Flags raised:**
- Unsigned subtraction without a prior bounds/sanity check.
- No guard that `hdr->size >= OSAL_MEM_HDRSZ` before the subtraction.
- Resulting corrupted free-list entry propagates silently.

## Plan

### 1. Read and understand the current code

- Read the full `osal_mem_free` function in `osal_memory.c`.
- Identify the exact line(s) where `size - OSAL_MEM_HDRSZ` (or equivalent) is computed.
- Note the types involved (`uint16_t`, `uint8_t`, etc.) and the header struct definition (`osalMemHdr_t` or similar).

### 2. Identify all unsafe subtractions

- Search for every subtraction of `OSAL_MEM_HDRSZ` (or the header size constant) from `hdr->size` or any block-size field within `osal_mem_free`.
- Also check the companion `osal_mem_alloc` to understand the invariants that are expected to hold.

### 3. Add a guard before the subtraction

Before the computed free size is used, insert a defensive check:

```c
if (hdr->size < OSAL_MEM_HDRSZ) {
    /* Heap corruption or invalid pointer — abort rather than corrupt the free list */
    HAL_ASSERT_FORCED();   /* or equivalent assert/trap for this platform */
    return;
}
```

This prevents the wraparound from occurring even if the heap is already corrupted, converting a silent memory corruption into a detectable fault.

### 4. Optionally cast to a signed intermediate

If a hard abort is too disruptive for the target environment, an alternative is to cast to a wider signed type before subtracting and check for negativity:

```c
int32_t freeSize = (int32_t)hdr->size - (int32_t)OSAL_MEM_HDRSZ;
if (freeSize <= 0) {
    HAL_ASSERT_FORCED();
    return;
}
```

Use whichever style is consistent with the rest of the OSAL codebase.

### 5. Verify no other callers depend on the wrapped value

- Search the file and related files for any place that reads the computed free size and might inadvertently rely on the current (broken) behavior.
- Check `osal_mem_merge` or equivalent coalesce helpers that walk the free list.

### 6. Test

- Run any existing OSAL heap unit tests.
- Add a test that deliberately passes a pointer whose header reports a `size` smaller than `OSAL_MEM_HDRSZ` and confirms that the function traps/returns cleanly rather than corrupting the free list.
- Verify that normal alloc/free cycles still pass.
