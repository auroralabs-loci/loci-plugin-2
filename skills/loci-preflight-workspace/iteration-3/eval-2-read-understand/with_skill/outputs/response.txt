`GAPBondMgr_FindAddr` is a function in the TI BLE5 stack's GAP Bond Manager that searches the bonding database for a stored bond record matching a given Bluetooth device address. It is used to look up whether a particular peer device has a pre-existing bond (i.e., has previously paired and exchanged long-term keys with the local device).

The function scans through the non-volatile (NV) storage entries where bond records are persisted and returns the index of the matching entry if found, or a sentinel value (typically an invalid index) if no match exists.

1. `pDevAddr` — A pointer to a 6-byte Bluetooth device address (BD_ADDR) to search for. This is the address of the peer device whose bond record you want to locate.

2. `addrType` — The address type of the peer device (e.g., public, random static, resolvable private, non-resolvable private). This is used together with `pDevAddr` to uniquely identify the peer, since the same 6-byte value can represent different devices depending on the address type.

3. `pIdx` — An output parameter (pointer to a uint8_t or similar index type) that is populated with the bond table index of the matching record if a bond is found. The caller can use this index to retrieve or update the full bond record.
