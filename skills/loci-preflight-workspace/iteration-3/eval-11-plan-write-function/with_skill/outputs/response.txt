## Adjusted Plan

**1. Confirm the handle type at the call site**
Read the signature of HCI_SendDataPkt in hci.c and its header to confirm the handle parameter type. If it is uint16_t, no cast is needed. If it is uint8_t (unlikely given BLE spec) or a signed type, add an explicit cast in the validator.

**2. Define the error code**
Check whether a HCI_ERROR_CODE_INVALID_HCI_CMD_PARAMS or equivalent already exists in the HCI error-code enum (likely in hci.h or bcomdef.h). Reuse an existing code rather than introducing a new one. If none fits, define HCI_ERROR_INVALID_CONN_HANDLE in the appropriate header as a step before the function edit.

**3. Write the validator function**

Place it in hci.c, above HCI_SendDataPkt, with a forward declaration in the corresponding header if it will be called from outside the file:

```c
/*
 * ValidateBLEConnHandle
 *
 * Returns HCI_SUCCESS (0) if handle is in the valid BLE connection-handle
 * range [0x0000, 0x0EFF] per Bluetooth Core Spec Vol 4, Part E §7.1.5.
 * Returns HCI_ERROR_INVALID_CONN_HANDLE otherwise.
 *
 * The parameter is uint16_t to match HCI_SendDataPkt's own handle type and
 * to avoid signed-to-unsigned promotion hazards at the call site.
 */
static uint8_t ValidateBLEConnHandle(uint16_t connHandle)
{
    /* Valid range: 0x0000–0x0EFF (Bluetooth Core Spec, Vol 4 Part E §7.7.5) */
    if (connHandle > 0x0EFFu)
    {
        return HCI_ERROR_INVALID_CONN_HANDLE; /* out-of-range handle */
    }
    return HCI_SUCCESS;
}
```

**4. Guard every call site of HCI_SendDataPkt**

Insert the guard at each point in hci.c (and any other files that call HCI_SendDataPkt directly) before the call is issued:

```c
uint8_t status = ValidateBLEConnHandle(connHandle);
if (status != HCI_SUCCESS)
{
    return status;
}
HCI_SendDataPkt(connHandle, pbFlag, pktLen, pData);
```

Do not guard only the new path — an unguarded existing call site defeats the validator.

**5. Add a unit test or assertion** (if the project has a test harness)
Verify that handles 0x0EFF, 0x0F00, 0x0000, and 0xFFFF all produce the expected return codes.
