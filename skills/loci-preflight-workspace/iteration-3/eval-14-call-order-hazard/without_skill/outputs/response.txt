## Change to `oad.c`

### 1. Add the required include (if not already present)

Near the top of `oad.c`, ensure `gapbondmgr.h` is included so `GAPBondMgr_FindAddr` is declared:

```c
#include "gapbondmgr.h"
```

### 2. Modify `OAD_open` to reject unbonded devices first

`OAD_open` receives a connection handle (or the peer address can be resolved from it). The call to `GAPBondMgr_FindAddr` should be inserted as the very first executable statement inside the function, before any resource allocation or service registration:

```c
bStatus_t OAD_open(uint16_t oadTimeout)
{
    uint8_t peerAddr[B_ADDR_LEN];
    uint8_t addrType;

    // Reject image transfers from unbonded devices before doing any setup.
    // GAPBondMgr_FindAddr returns SUCCESS only if the peer is in the bond table.
    if (GAPBondMgr_FindAddr(peerAddr, addrType, NULL) != SUCCESS)
    {
        return OAD_NO_RESOURCES;   // or a more specific error code such as bleNotConnected / FAILURE
    }

    // ... rest of the existing OAD_open body (alloc, mq_open, clock init, etc.)
}
```

> **Note on obtaining the peer address:** `OAD_open` in the cc26xx profile does not currently receive the peer's BLE address as a parameter — it is called at the application level when an OAD write to the image identify characteristic is received. The peer address and address type should be extracted from the connection event data (e.g., via `GAP_GetDevAddress` or from the `linkDB` entry for the active connection handle) before passing them to `GAPBondMgr_FindAddr`. If there is already a connection-handle-to-address lookup helper in your application (such as `Connection_getAddr`), use that here.

### 3. Choose an appropriate error return

The current `OAD_open` returns `OAD_SUCCESS` (0) or `OAD_NO_RESOURCES`. If there is a more semantically appropriate code for "peer not bonded" (e.g., `bleNotConnected`, `FAILURE`, or a custom `OAD_NOT_BONDED` define), prefer that so callers can distinguish the rejection reason.

---

## Summary of changes

| Location | Change |
|---|---|
| `oad.c` — includes | Add `#include "gapbondmgr.h"` if not already present |
| `OAD_open` — first statement | Call `GAPBondMgr_FindAddr`; return early if peer is not bonded |
| Caller / application layer | Ensure peer address and address type are available before `OAD_open` is called |

This ensures no OAD resources (queue, CCCD table, GATT service slot, clock) are allocated for a connection that will be rejected anyway, and that unbonded devices cannot initiate a firmware update.
