## Plan: Add safe_divide(uint32_t a, uint32_t b)

### Goal
Implement a function `safe_divide(uint32_t a, uint32_t b)` that returns `a / b`, or `0` if `b` is zero.

---

### Step 1 — Decide where the function lives

Determine the appropriate header and source file. Options:
- If a general math/utility module already exists (e.g., `math_utils.h` / `math_utils.c`), add it there.
- Otherwise, create a new `safe_math.h` / `safe_math.c` pair under the relevant source directory.

---

### Step 2 — Write the header declaration

In the chosen `.h` file, add:

```c
#include <stdint.h>

/**
 * @brief Divide a by b, returning 0 if b is zero.
 *
 * @param a  Dividend (unsigned 32-bit).
 * @param b  Divisor  (unsigned 32-bit).
 * @return   a / b, or 0 if b == 0.
 */
uint32_t safe_divide(uint32_t a, uint32_t b);
```

---

### Step 3 — Write the implementation

In the matching `.c` file:

```c
#include "safe_math.h"   /* or whichever header is chosen */

uint32_t safe_divide(uint32_t a, uint32_t b)
{
    if (b == 0U) {
        return 0U;
    }
    return a / b;
}
```

Notes:
- Both operands are `uint32_t`, so no sign-extension or negative-divisor concerns.
- Integer division of two unsigned 32-bit values cannot overflow, so the only guard needed is the zero-divisor check.
- The `U` suffix on literals keeps the expression unsigned throughout, avoiding implicit promotions.

---

### Step 4 — Add a unit test

Write at least three test cases:
1. Normal case: `safe_divide(10, 2)` == 5
2. Zero divisor: `safe_divide(42, 0)` == 0
3. Zero dividend: `safe_divide(0, 5)` == 0
4. Division that truncates: `safe_divide(7, 2)` == 3

---

### Step 5 — Integrate into the build

- Add the new `.c` file to the relevant `CMakeLists.txt`, `Makefile`, or build script.
- Include the new header wherever `safe_divide` will be called.

---

### Risks and considerations

- None specific to arithmetic: unsigned integer division is well-defined in C for all non-zero divisors.
- The only runtime risk is the divide-by-zero case, which is explicitly handled.
- No dynamic allocation, no ISR interaction, no external resources — this function is safe to call from any context.

---

### Summary

This is a straightforward two-branch function. The implementation is three lines of code. The main decisions are file placement and naming convention, both of which should follow the project's existing patterns.
