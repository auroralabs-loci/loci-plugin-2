## Plan: Add `safe_divide(uint32_t a, uint32_t b)`

The task is to add a function `safe_divide(uint32_t a, uint32_t b)` that returns `a / b`, or `0` if `b` is zero.

## Edit steps

1. **Declare the function** in the appropriate header (e.g. `math_utils.h`
   or whichever translation unit header covers arithmetic helpers):

   ```c
   uint32_t safe_divide(uint32_t a, uint32_t b);
   ```

2. **Define the function** in the corresponding `.c` file:

   ```c
   uint32_t safe_divide(uint32_t a, uint32_t b) {
       if (b == 0u) {
           return 0u;
       }
       return a / b;
   }
   ```

   Implementation notes:
   - The guard `b == 0u` is evaluated before the division; the division
     expression is never reached when `b` is zero, so no undefined behaviour
     or hardware trap can occur.
   - Using `0u` and the `uint32_t` return type keeps every value in the
     expression unsigned, avoiding implicit promotion surprises.
   - The function is a pure leaf: it reads no globals, allocates no
     resources, and has no side effects, making it safe to call from any
     context (ISR, RTOS task, static initializer, etc.) without additional
     locking.

3. **Include the header** at any call site that uses `safe_divide`.

No further changes are required.
