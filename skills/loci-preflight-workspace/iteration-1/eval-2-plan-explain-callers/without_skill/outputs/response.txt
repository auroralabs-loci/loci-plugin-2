## What `connection_find` Does

`connection_find` is defined in `evals/preflight/fixtures/conn_manager.c`. Its signature is:

```c
ble_conn_t *connection_find(uint16_t handle)
```

It performs a **linear scan** over a fixed-size static table of active BLE connections (`s_conn_table`, capacity `MAX_CONNECTIONS = 8`). For each slot it checks whether the slot's `handle` field matches the requested `handle` and that the slot is marked `active`. If a match is found it returns a pointer to that `ble_conn_t` entry; otherwise it returns `NULL`.

In short: given a BLE connection handle, it looks up and returns the corresponding connection record, or `NULL` if no active connection with that handle exists.
